# Прошивка микроконтроллера

## Описание прошивки

Прошивка написана на языке C для AVR (IDE: AVRStudio 6). Автор оригинального кода: Vassilis Serasidis, лицензия GNU GPL v3.

**Целевой МК:** ATtiny13 при частоте внутреннего RC-генератора 9,6 МГц.

### Назначение выводов

| Вывод ATtiny13 | Обозначение | Функция |
|---|---|---|
| PB0 | COIL | Управление катушкой (выход) |
| PB1 | SIGNAL_IN | Приём сигнала с катушки (вход, прерывание PCINT1) |
| PB2 | BUZZER | Зуммер — сигнал успешного считывания (выход) |
| PB3 | PWM_PIN1 | Вспомогательный ШИМ-выход |
| PB4 | TXD | Передача ID метки по UART 9600 бод (выход) |

### Принцип работы

1. **Генерация поля 125 кГц.** Таймер Timer0 настроен в режиме Fast PWM с делителем 1 и значением сравнения OCR0A = OCR0B = 38. При тактовой частоте 9,6 МГц это даёт частоту переключения ≈ 125 кГц на выводе PB0 — несущая частота для активации RFID-меток EM4100.

2. **Приём и детектирование сигнала.** Прерывание PCINT0 фиксирует изменение уровня на PB1 (принятый сигнал с метки). Прерывание по переполнению таймера TIM0_OVF отслеживает длительность импульса и определяет текущий логический уровень (UP/DOWN).

3. **Декодирование EM4100.** Протокол EM4100 передаёт 64 бита в кодировке Manchester. Функция `readTagSerialNumber()` читает 10 групп по 5 бит (4 бита данных + 1 бит чётности строки), проверяет чётность строк и 4 столбцов. При ошибке чётности — считывание отбрасывается.

4. **Передача ID.** При успешном считывании (parity == TRUE, сумма байт > 0) функция `hex2Ascii()` преобразует 32-битный номер в десятичную ASCII-строку и побитно передаёт её через PB4 программным UART (9600 бод, 8N1). Затем зуммер подаёт сигнал 500 мс.

### Ключевые константы прошивки

```c
#define F_CPU     9600000UL  // Тактовая частота ATtiny13
#define PWM_VALUE 38         // OCR0A/OCR0B → частота ~125 кГц
#define BAUDRATE  9600       // Скорость UART (бод)
#define ONE_BIT_DELAY (1000000/BAUDRATE)  // Задержка одного бита UART = 104 мкс
```

## Загрузка прошивки

### Необходимое оборудование и ПО

- Программатор **USBasp** с ISP-кабелем (или аналог — USBtinyISP, AVRISP mkII);
- ПК с установленной утилитой **avrdude**;
- файл прошивки `RFID_Reader_v1.hex`.

### Порядок прошивки (операция 060 технологической карты)

**Шаг 1.** Подключить программатор USBasp к разъёму ISP на плате. Питание платы — от программатора или от внешнего источника 5 В.

**Шаг 2.** Записать прошивку:

```bash
avrdude -c usbasp -p attiny13 -U flash:w:RFID_Reader_v1.hex
```

Успешный результат: `avrdude: verifying ... OK`.

**Шаг 3.** Проверить Fuse-биты (должны быть установлены при первой прошивке):

```bash
# Чтение текущих Fuse-битов
avrdude -c usbasp -p attiny13 -U lfuse:r:-:h -U hfuse:r:-:h
```

Требуемые значения:

| Fuse | Значение | Назначение |
|---|---|---|
| CKSEL | внутренний RC 9,6 МГц | Источник тактирования |
| BODLEVEL | 1,8 В | Защита от пониженного питания |

**Шаг 4.** Отключить программатор. Подать питание 9 В от батареи и выполнить функциональный контроль (операция 065).

!!! note "Примечание"
    ATtiny13 установлен в панель DIP-8. При необходимости перепрошивки МК можно извлечь из панели и подключить к программатору отдельно, не вскрывая корпус полностью.

## Компиляция из исходного кода

Исходный файл: `125kHz_RFID_reader.c`.

```bash
# Компиляция
avr-gcc -mmcu=attiny13 -DF_CPU=9600000UL -Os -o rfid.elf 125kHz_RFID_reader.c

# Создание HEX-файла
avr-objcopy -O ihex rfid.elf RFID_Reader_v1.hex
```

IDE: AVRStudio 6 (выбрать устройство ATtiny13 или ATtiny13A перед компиляцией — частота и значение PWM_VALUE выбираются автоматически через директивы препроцессора).
